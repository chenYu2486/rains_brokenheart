                       中文摘要

    摘要：随着公众心理健康需求不断上升，传统心理咨询在服务成本、可及性、

隐私保护和早期筛查方面仍存在明显限制。大语言模型具备自然语言理解、多轮对

话和共情表达能力，为低门槛心理支持系统提供了新的技术路径；但通用模型直接

用于心理健康场景时，仍可能出现幻觉、状态追踪不足、风险识别不稳和伦理边界

模糊等问题。本文以“基于大模型微调的精神心理对话系统开发”为题，结合开题

报告、项目中期汇报和实际源码，设计并实现了一套面向心理健康初筛与陪伴支持

的端到端原型系统。系统采用评估模型与陪伴模型分阶段协作的双模型架构：评估

模型负责建档访谈、周期复评和结构化画像生成，陪伴模型基于最新画像开展持续

对话；同时引入本地检索增强生成模块，将 DSM-5-TR 相关知识切分为可追溯片

段，在涉及专业内容时为回答提供来源依据。工程实现方面，系统使用原生

JavaScript、HTML 和 Tailwind CSS 完成交互界面、状态机、流式 API 调用、会话存

档、画像面板和本地 RAG 检索闭环，并预留模型 ID 配置能力，便于接入 LoRA/SF

T 微调后的模型。测试与分析表明，系统能够完成关键词定制、初评建档、JSON 画

像生成、持续陪伴、周期复评、知识库命中回显和异常提示等核心流程，具有一定

可解释性、可扩展性和工程复现价值。

                                     ABSTRACT
     ABSTRACT: With the growing demand for mental health support, traditional
counseling services still face limitations in cost, accessibility, privacy protection, and early
screening. Large language models provide a new technical path for low-threshold
psychological support because they can understand natural language, sustain multi-turn
dialogue, and generate empathetic responses. However, directly applying general-purpose
models in mental health scenarios may introduce hallucinations, weak state tracking,
unstable risk awareness, and unclear ethical boundaries. This thesis designs and
implements an end-to-end prototype system for mental health pre-screening and supportive
dialogue based on the proposal, interim presentation, and project source code. The system
adopts a staged dual-model architecture: the assessment model handles intake interviews,
periodic reassessment, and structured profile generation, while the companion model
provides ongoing supportive dialogue based on the latest profile. A local retrieval-
augmented generation module is also integrated to split DSM-5-TR knowledge into
traceable chunks and provide evidence when professional content is involved. The
implementation uses native JavaScript, HTML, and Tailwind CSS to build the user
interface, state machine, streaming API calls, conversation archive, assessment dashboard,
and local RAG pipeline. Configurable model identifiers are reserved for future LoRA/SFT
fine-tuned models. Functional analysis shows that the system supports keyword
customization, intake dialogue, JSON-based assessment, continuous companionship,
periodic reassessment, knowledge citation display, and error handling, demonstrating
interpretability, extensibility, and engineering reproducibility.

KEYWORDS: large language model; mental health dialogue system; LoRA fine-tuning;
retrieval-augmented generation; structured assessment

1 引言

1.1 研究背景与意义

  精神障碍从个人问题上升为公共健康及社会治理的重大问题已成为共识，《世

界精神卫生报告》强调目前全球范围内精神卫生服务存在巨大需求量、覆盖率低以

及社会偏见等因素限制了精神卫生服务发展等问题[1]。“健康中国行动(2019—2030

年)”中也提出了我国居民心理健康素养提升计划、建立心理危机干预和心理援助服

务模式的目标[2]。这意味着，心理健康服务不再是单纯的医疗卫生系统内部专业问

题，而是一个教育、社区乃至互联网服务及智能化相关的问题。

  人工咨询师一对一的服务具备专业度高、界限感强等特点，但也面临价格昂

贵、等待时间长、地区不平衡以及求询者门槛高等客观制约因素。对一些处于问题

萌芽阶段、情绪不稳定或暂不具备实地咨询的群体来说，一款价格低廉、可以随时

使用、提供初级聆听与预警服务的软件不失为一种现实选择。它可以不代替医生、

心理咨询师，但可成为早期筛查、情绪支持和求助引导的入口。

  近期大语言模型在语义理解和文本生成、指令跟随、多轮对话等方面得到了长

足的发展。相较于早期基于规则或者模板的聊天机器人而言，大模型可以理解用户

较为复杂的描述，并能够在开放式的对话环境下产生相对自然的回答。这同样启发

了我们对心理对话系统建设的新想法：以用户主诉的症状、情绪强度、睡眠情况、

人际关系以及风险线索等逐渐建立档案并随时根据下文进行协助。

  然而，心理健康场景是高度敏感的高危场景。通用大模型存在事实幻觉、过度

建议、漏掉危机信号、隐私泄露及表达不当等风险。如果模型给出未经验证的诊断

或处方可能会对用户产生误导。因此，本课题不求系统替代专业诊疗，只做“心理

状态初步筛查和支持式聊天原型”。通过模型分工、结构化评价、本地知识库搜

索、风险预警来规避使用风险。

1.2 国内外研究现状

  AI 在心理健康领域并不是一蹴而就的。早期 ELIZA 的模式匹配模仿罗杰斯式

咨询语言实现了人机对话的情绪陪伴作用，本质上是基于规则的替换，并不具有真

正意义上的语义理解 [9] 。之后利用机器学习以及深度学习进行情感分类、抑郁倾

向识别及社交媒体的心理状态分析，模型的能力逐渐从浅层特征工程向语义表示学

习发展。

  随着大语言模型的到来，关注点由单篇文本分类拓展到了开放式的心理健康问

答、可解释分析及对话干预上。例如，Mental-LLM 等工作对各类指令微调模型进行

了在线文本心理健康预测任务测试，证明大模型在零样本、少样本环境下具有一定

的可行性，但同时受制于数据范围、文化和任务定义上的约束 [5]。MentaLLaMA 和

可解释心理健康分析的相关工作指出心理健康 AI 需要不仅提供标签，还需说明判

断的原因以提高可解释性及对决策的价值[6][7]。

  在专业性准确性上，检索增强生成(RAG) 被用来解决大模型幻觉的问题，在生

成之前检索外部的知识库，并把权威的片段当作 Context 输入给模型，使得答案会

受到知识边界的限制[4]。在心理领域里，当涉及诊断标准、症状的解释、鉴别诊

断、危机干预的原则等内容时，RAG 是很适用的一种技术手段。但是 RAG 同样存

在检索噪声、上下文拼接生硬以及多轮会话中的知识更新不稳定性的问题。

  在模型适配上，LoRA 等 PEFT 范式下可以将基座模型的权重冻结住，只对低

秩的适配矩阵进行训练便能完成垂直领域的微调，极大节省了垂直领域微调的算力

开销[3]。对于高校毕业设计及小批量试验场景而言，LoRA 可提供一种可复用的途

径来使用相对较小的标注样本提升模型的风格表达性、指令遵循性及领域任务性

能，然后 RAG 补足知识正确性。这样，微调解决 “怎么” 和 “按照怎样的任务

格式输出” 的问题，RAG 解决 “根据什么来” 的问题，两者是互补的关系。

  现有研究虽然已经覆盖提示工程、RAG、微调和可解释分析等方向，但在面向

真实使用流程的系统集成方面仍有不足。多数论文关注单一任务，例如抑郁检测、

风险分类或知识问答，较少给出一个完整的“关键词定制、初评建档、动态画像、

2 相关技术基础

2.1 大语言模型与心理健康对话

  LLM 通常使用 Transformer 在大量的文本上做预训练，并通过指令微调、偏好

对齐等方式来增强对用户指令的服从程度，在传统任务型系统的优点基础上增加了

可以接收开放域的输入信息、理解上下文及生成自然流畅回答的能力。心理健康对

话恰好具有开放性高、歧义性高、情感强度高及上下文相关度高等特点，适合于基

于大模型进行交互。

  但是心理健康的聊天又不等于闲聊。用户可能描述失眠、持续性的焦虑情绪、

自我贬低、创伤事件甚至是自伤的想法。模型不仅要给予语言层面的理解和支持，

同时也要注意不过度诊断、不要提供用药指导，在发现危险信号的时候首先引导用

户寻求现实世界中可信赖的人以及专业的帮助。所以，系统设计上必须对提示词、

状态机、风险提示以及知识来源进行约束。

2.2 LoRA 参数高效微调

  全量微调大模型会更新全部参数，训练成本很高并且很容易破坏基座模型的通

用能力。LoRA 的主要思想是在不更新原有权重的情况下，对注意力或者前馈网络

中的线性层加入低秩矩阵，同时将权重的更新量近似为两个较小矩阵相乘的形式。

若原权重矩阵为 W，则微调后得到的新权重可以表示成 W+BA 的形式，其中 A 和

B 是可学习的低秩矩阵。因为可学习参数大大减少，所以 LoRA 适用于资源受限条

件下的垂域适配 [3]。

  在本课题里，LoRA 微调的目的不是训练一个固定的答案，而是希望模型能学

会心理支持对话里的表达风格、结构化信息提取格式和安全拒答策略。即微调的数

据应该包括用户的自述、咨询师的回答、评估字段抽取、危机提示以及知识不足时

候的保守回答。使用新微调后的模型只需在系统设置中更改模型 ID 即可，在不修

改前端框架的情况下就可以实现。

2.3 检索增强生成技术

   RAG 使用外部的知识源辅助语言模型进行推理。通常步骤为文档处理、分段、

编码或者关键词存储、相似性搜索、片段组合以及答案合成。相对于纯参数化的方

法，RAG 可以直接利用可信的信息来源，在一定程度上缓解模型在特定领域的虚假

信息问题，并且给出其出处 [4]。

   本系统所使用的 RAG 实现为一个较轻量化的浏览器端实现，在知识库中以

document 的 meta data 和 chunks 数组的形式存储了 chunk 中的源信息、页面区间、

chunk id、chunk 正文内容的 hash 值以及 chunk 正文等内容；在浏览器端装载好知

识库之后，将用户的问题和各个知识库中的知识片断进行 tokenize 后，生成其哈希

值，并计算相似性分数，然后再根据综合得分取 TopK 片段注入系统提示词。此方

案不需要依赖后端向量数据库，方便做毕业设计展示和静态部署。

2.4 流式 API 与前端状态机

   心理对话系统需要较好的实时性，在等待模型完整生成后才显示的话会让用户

感觉交互很慢。流式的 API 通常是 Server-Sent Events 或者类似的协议一步步返回

增量文本，前端一边接受一边渲染可以模拟出自然的打字效果。项目中 api.js 封装

了非流式的 chat 和流式的 streamChat 两种请求，包含超时机制、异常捕获、重试

策略。

   除通信层外，系统还需要显式状态机来控制对话阶段。本项目将会话划分为

idle、intake 和 therapy 三个状态。idle 表示尚未开始；intake 阶段只调用评估模型；

therapy 阶段主要调用陪伴模型，但在达到复评轮次后会插入一次评估模型调用。这

样的状态机使系统行为可解释、可测试，也便于在界面上显示当前模型身份。

2.5 安全伦理与应用边界

   精神心理领域使用的大语言模型要有边界感。第一是系统输出仅能作为辅助性

4.1 关键词定制与建档入口

    系统预置 20 个常见心理议题关键词，包括原生家庭、亲密关系、职场内耗、存

上。这样做的好处在于不会剥夺用户的主动权，在开始阶段也不会让用户有面对量

表式的压迫感。

6.3 关键流程分析

    从测试结果看，系统的核心流程能够闭环运行。初评阶段的关键是 intakeTurns

计数与评估模型调用保持一致；陪伴阶段的关键是 therapyTurnsSinceReview 触发复

评后必须重建系统提示词，否则陪伴模型无法感知最新画像。项目代码在

generateAssessment   完   成   后   重   置    therapyTurnsSinceReview   ，   并    在

rebuildTherapyContext 中写入 latestReport，保证状态更新能够影响后续回复。

    RAG 流程的优势在于轻量、可离线加载和可解释。当前实现不依赖 FAISS 等后

端向量库，而是在浏览器端用哈希向量和词项重叠计算相关度。这种方式便于演示

和部署，但语义表达能力弱于真正的嵌入模型。对于毕业设计原型而言，它已经能

够展示知识库切块、检索、提示注入和引用回显的完整链路；如果用于实际生产，

则应该用 bge-m3 这类嵌入模型加上重排序器替代。

    双模型架构的运行过程也具有较强可观察性。每条助手消息和每份评估报告都

记录 model、role、stage 和 stageLabel，前端能够直接显示当前由哪个模型承担回

复。相比只在设置面板保留两个模型输入框，这类元数据记录更能说明模型在运行

6.4 系统优势

 流程闭环较完整：关键词选择、建档访谈、画像生成、陪伴回复和复评更新由同
  一状态机串联，演示路径连续。
 模型分工较明确：评估模型负责信息采样与结构化输出，陪伴模型负责支持性回
  应，减少单一提示词同时承担多种角色造成的混乱。
 知识来源可以回看：本地 RAG 保留页码、片段编号和匹配分数，涉及知识库内
  容的回答能够显示引用依据。
 运行部署较轻量：前端基于原生 JavaScript 和静态文件实现，不依赖复杂后端，
  便于毕业设计展示、调试和迁移。
 扩展接口较充足：API Base、模型 ID、建档轮次、复评间隔和 RAG 阈值均可配
  置，后续接入微调模型或后端服务成本较低。

6.5 局限性分析

  一是当前项目仓库主要体现前端原型和本地 RAG 链路，微调训练过程与 LoRA

权重尚未直接封装在仓库中。因此，本文将微调部分作为系统方法与接入方案进行

说明，而不声称前端仓库已包含完整训练流水线。后续应补充训练脚本、数据版

本、loss 曲线、评测日志和模型权重管理说明。

  二是本地 RAG 基于哈希向量和词项重叠，可以满足轻量演示，但对同义表达和

复杂语义关系的召回能力有限。心理健康用户常使用模糊表达，如“心里堵”“脑

子里不停转圈”“什么都不想动”，简单词项检索未必能稳定命中专业知识片段。

  第三，系统当前使用 localStorage 保存会话，对隐私保护不够充分。真实部署需

要引入加密存储、用户授权、自动过期、脱敏导出和数据删除机制。最后，系统的

安全策略仍主要依赖提示词和风险字段，面对复杂危机个案时仍可能不足，必须强

调人工专业支持和线下求助。

7 结论与展望

7.1 研究结论

  本文围绕“基于大模型微调的精神心理对话系统开发”完成了需求分析、总体

设计、主要模块实现描述、微调与知识库设计方案和功能测试分析。系统将大语言

模型应用到心理健康初筛和支持性对话场景中，并通过评估模型与陪伴模型分阶段

协作，缓解传统聊天系统缺乏结构化跟踪和模型角色混淆的问题。

  项目源码表明，系统已实现关键词定制、建档访谈、评估画像、持续陪伴、周

期复评、本地知识库检索、流式 API、会话存档和模型阶段可视化等核心功能。

RAG 模块将 DSM-5-TR 相关资料转换为带页码和哈希的知识片段，并在检索命中后

注入提示词，让专业性的答案也具有一定的溯源依据。同时，系统也预留了模型设

置的能力，这将为后续 LoRA 或者 SFT 的微调提供工程上的可能。

  在毕业设计的角度下，本文的价值主要体现在将开题报告中提出的技术路线 “

微调 + RAG + 要素化采集 ” 落地为一个可以运行、解释、扩展的端到端的原型

中，在仍然无法代替专业心理咨询以及医学诊断的前提下，作为早期的情绪支持、

问题梳理和风险提示工具有一定的探索意义。

7.2 后续展望

  后续工作可从四个方面展开：一是补全完整微调实验，形成数据清洗、 LoRA

训练、自动评测、人工抽检和消融实验闭环，并接入现有双模型架构；二是升级

RAG 检索能力，引入语义嵌入模型、向量数据库和重排序器，提高对中文口语化表

达的召回能力；三是强化安全机制，在输入端、生成端和输出端分别加入风险识

别、危机模板、人工转介和输出审核；四是完善隐私与部署方案，加入本地加密、

数据脱敏、权限管理和用户知情同意机制。

  此外，系统还可以扩展移动端适配、长期记忆摘要、用户主动删除记录、心理

量表辅助填写和咨询师后台查看等功能。但无论技术如何发展，心理健康 AI 都应坚
